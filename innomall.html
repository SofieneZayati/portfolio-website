Project Overview

InnoMall is a comprehensive mall management system that bridges the gap between traditional retail management and modern technology. This integrated solution combines a powerful JavaFX desktop application for administrative tasks with a sophisticated PHP Symfony web interface for customer interactions. The system incorporates cutting-edge features including IoT-based parking management, AI-powered customer support, and seamless payment processing.

Designed to revolutionize the shopping center experience, InnoMall provides real-time analytics, automated operations, and enhanced customer engagement tools. The system was developed using an agile methodology over a 6-month period, with iterative testing and feedback from mall operators and shoppers.

Technical Architecture

Backend Technologies

  • PHP Symfony 6 - Web application framework
  • JavaFX - Desktop application interface
  • MySQL - Primary database management
  • RESTful API - Service communication layer
  • Doctrine ORM - Database abstraction layer
  • Twig - Templating engine for web interface

IoT & Hardware Integration

  • ESP32 microcontrollers for sensor management
  • Ultrasonic sensors for parking space detection
  • MQTT protocol for real-time data transmission
  • WiFi connectivity with automatic reconnection
  • BLE beacons for indoor navigation
  • Failover to Ethernet when WiFi is unstable

External Integrations

  • Stripe API - Payment processing and management
  • Gemini API - AI-powered chatbot functionality
  • Twilio API - SMS and communication services
  • Google Maps API - Location and navigation services
  • Firebase - Real-time database for mobile sync
  • JasperReports - Advanced reporting engine

Development Challenges & Solutions

Developing InnoMall presented several technical challenges that required innovative solutions. The most significant was ensuring real-time synchronization between the IoT parking sensors, desktop admin interface, and customer web portal. We implemented an MQTT-based pub/sub system with a WebSocket fallback to maintain data consistency across all platforms.

Key Challenges Addressed

  • Real-time data synchronization between IoT devices and web platform using MQTT with Redis caching
  • Secure payment processing with PCI compliance and tokenization through Stripe
  • Scalable architecture design using microservices for high-traffic mall environments
  • Cross-platform compatibility between JavaFX desktop and Symfony web interfaces
  • AI chatbot integration with context-aware responses using Gemini API
  • Offline functionality for parking system with automatic sync when connection resumes
